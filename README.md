## What is this?

[StartOver.xyz](https://startover.xyz) is a free-to-play, massively-multiplayer personal transformation game built on the thoughtware of **Possibility Management** — a framework for next culture, conscious relating, emotional intelligence, and regenerative living created by Clinton Callahan.

Over the years, the community built a cloud of **700+ websites** on Strikingly, speaheaded by [Clinton Callahan](https://www.clintoncallahan.org/) — each one covering a concept, experiment, tool, or gameworld within the Possibility Management ecosystem. 

Topics range from archetypal anger and bright principles to radical responsibility, adulthood initiations, love-building practices, and dozens of intentional community projects around the world.

In April 2026, Strikingly took down all of these sites simultaneously. This project recovers them from [archive.org](https://archive.org) (the Wayback Machine) and hosts them as static sites under the `startover.world` domain — so that `2dramas.startover.world`, `10doorways.startover.world`, `radicalresponsibility.startover.world` and 700 others are accessible again.

The root domain [startover.world](https://startover.world) serves as a **Spaceport** — a searchable directory of all recovered sites.

## Scripts

### `crawl.mjs` — Wayback Machine HTML Crawler
Queries the CDX API for the latest archived snapshot of each domain, fetches the raw HTML, strips the Wayback toolbar injection, and saves it to `sites/<slug>/index.html`.

```bash
# Normal run (resumes where it left off)
node crawl.mjs all_links.md

# Refresh mode — checks for newer snapshots and re-crawls only updated ones
node crawl.mjs all_links.md --refresh
```

- Saves progress to `crawl-progress.json` after every domain — safe to stop and restart
- Logs everything to `crawl.log`
- Delay of 2 seconds between requests to be respectful to archive.org
- 3 retries per domain with exponential backoff

---

### `mirror.mjs` — Full Asset Mirror
Uses `wget --mirror` to fetch the complete site — HTML, CSS, JavaScript, images, fonts. Run this after `crawl.mjs` for production-quality results. Requires `wget` installed.

```bash
brew install wget       # macOS
node mirror.mjs all_links.md
```

Run this overnight — it takes 1–3 minutes per site.

---

### `build-index.mjs` — Generate the Spaceport
Reads `crawl-progress.json` and `all_links.md` and generates `index.html` — the searchable landing page at `startover.world`. Re-run this after each crawl pass to update the site count.

```bash
node build-index.mjs all_links.md
# Then deploy:
scp index.html hostinger:/var/www/startover/index.html
```

---

### `serve.mjs` — Local Dev Server
Runs a local server that mimics the production routing — subdomain-based, just like the live server.

```bash
node serve.mjs
```

- `http://localhost:3000` → Spaceport index
- `http://2dramas.localhost:3000` → preview a specific site
- `http://random.localhost:3000` → notfound page

Works natively in Chrome and Firefox on macOS — no `/etc/hosts` edits needed.

## Contributing

The main ways to contribute to this project:

**1. Add missing sites**
If you know of a Strikingly site from the StartOver/Possibility Management ecosystem not in `all_links.md`, add it as a line in the format:
```
https://sitename.mystrikingly.com — Title of the site
```

**2. Fix broken recoveries**
Some sites came back with old snapshots or broken HTML. Check `crawl-progress.json` for entries with `status: error` or check `_meta.json` timestamps. You can manually supply a better archive URL or flag it for re-crawl.

**3. Run the mirror pass**
The current recovery is HTML-only. Running `mirror.mjs` on all successful sites will recover full CSS, images, and JS — making the sites look as they originally did. This takes significant time and bandwidth.

**4. Improve the Spaceport**
The `build-index.mjs` script generates `index.html`. Improvements to search, filtering by category, or better descriptions are welcome.

## Troubleshooting

**Crawler keeps timing out**
Increase `DELAY_MS` in `crawl.mjs` to `3000` or more. archive.org throttles heavy traffic.

**Old snapshot being fetched**
The CDX API query uses `limit=-1` to get the newest snapshot. Verify a domain manually:
```
http://web.archive.org/cdx/search/cdx?url=DOMAIN&output=json&limit=-1&fl=timestamp,original,statuscode&filter=statuscode:200
```

**Site shows broken layout**
HTML-only recovery strips styles. Run `mirror.mjs` for full asset recovery.

**nginx 500 on unknown subdomain**
The `notfound.html` file should be at `/var/www/startover/notfound.html` and the nginx error_page directive should be in place. See `SETUP.md`.

**SSL expired**
See SSL renewal instructions in the Infrastructure section above.
